VitePress 组件与数据：文章元数据获取与组件注册

VitePress

基于 CRUDClass Blog 实际项目，详细讲解 VitePress 中如何获取文章元数据信息，以及组件的创建、自动注册和引用方式

标签：
Vue 3 组件注册 createContentLoader
发布于 2025年12月19日
更新于 2026年3月16日

概述 ​

在 VitePress 博客项目中，我们主要关注两个核心能力：

  1. 获取文章元数据：通过 createContentLoader 获取所有文章的信息
  2. 组件注册与引用：自动注册组件并在 Markdown 中使用

文章元数据获取 ​

createContentLoader ​

VitePress 提供了 createContentLoader API 用于在构建时获取所有文章数据。

typescript
// .vitepress/content-loader.data.ts
import { createContentLoader } from 'vitepress'

export default createContentLoader('**/*.md', {
  includeSrc: true, // 包含原始 Markdown 源码
  render: true, // 包含渲染后的 HTML
  excerpt: true, // 包含摘要
  transform(raw): Post[] {
    return raw
      .filter(({ url }) => !url.includes('[') && !url.includes(']'))
      .map(({ url, frontmatter, excerpt, html }) => ({
        title: frontmatter.title,
        url,
        excerpt,
        html,
        date: formatDate(frontmatter.date),
        frontmatter: {
          title: frontmatter.title,
          cover: frontmatter.cover,
          category: frontmatter.category,
          tag: frontmatter.tag,
          description: frontmatter.description,
          ai: frontmatter.ai ?? false,
        },
      }))
      .sort((a, b) => b.date.time - a.date.time)
  },
})

数据类型定义 ​

typescript
export type PostDate = {
  time: number // 时间戳
  string: string // 格式化日期字符串
}

export type Post = {
  title: string
  url: string
  date: PostDate
  excerpt?: string
  frontmatter: Frontmatter
  html?: string
}

export type Frontmatter = {
  title: string
  category?: string[]
  tag?: string[]
  cover?: string
  date?: PostDate
  updated?: PostDate
  description?: string
  ai?: boolean
}

日期格式化 ​

typescript
function formatDate(raw: string): Post['date'] {
  const date = new Date(raw)
  date.setUTCHours(8)
  return {
    time: +date,
    string: date.toLocaleDateString('zh-CN', {
      year: 'numeric',
      month: 'long',
      day: 'numeric',
    }),
  }
}

组件自动注册 ​

主题入口配置 ​

typescript
// .vitepress/theme/index.ts
import type { Theme } from 'vitepress'
import DefaultTheme from 'vitepress/theme'
import { App, Component } from 'vue'
import Layout from './Layout.vue'
import './styles/main.css'

export default {
  extends: DefaultTheme,
  Layout: Layout,
  enhanceApp({ app }) {
    registerComponents(app)
  },
} satisfies Theme

function registerComponents(app: App): void {
  const components = import.meta.glob('./components/*.vue', { eager: true })
  for (const [path, component] of Object.entries(components)) {
    const componentName = path.replace('./components/', '').replace('.vue', '').replace(/\//g, '-')
    app.component(componentName, (component as { default: Component }).default)
  }
}

导出文章数据 ​

typescript
// .vitepress/theme/index.ts
import { data } from '../content-loader.data.ts'

export const articles = data

组件中使用文章数据 ​

vue
<script setup lang="ts">

示例：文章归档组件 ​

vue
<script setup lang="ts">
  import { computed, ref } from 'vue'
  import { articles } from '@theme/index.ts'

  const searchQuery = ref('')

  // 过滤有效文章
  const validArticles = computed(() => {
    return articles.filter((post) => {
      if (!post.title?.trim()) return false
      const layout = post.frontmatter.layout
      if (layout === 'home' || layout === 'page') return false
      return true
    })
  })

  // 搜索过滤
  const filteredPosts = computed(() => {
    if (!searchQuery.value.trim()) return validArticles.value

    const query = searchQuery.value.toLowerCase()
    return validArticles.value.filter(
      (post) =>
        post.title.toLowerCase().includes(query) ||
        post.frontmatter.category?.some((cat) => cat.toLowerCase().includes(query)) ||
        post.frontmatter.tag?.some((tag) => tag.toLowerCase().includes(query))
    )
  })
</script>

访问文章字段 ​

vue
<template>
  <article v-for="post in articles" :key="post.url">
    <h2>{{ post.title }}</h2>
    <time>{{ post.date.string }}</time>
    <p>{{ post.frontmatter.description }}</p>
    <div class="tags">
      <span v-for="tag in post.frontmatter.tag" :key="tag">{{ tag }}</span>
    </div>
    <div class="categories">
      <span v-for="cat in post.frontmatter.category" :key="cat">{{ cat }}</span>
    </div>
  </article>
</template>

组件创建与引用 ​

创建新组件 ​

在 .vitepress/theme/components/ 目录下创建 Vue 组件：

vue
<!-- .vitepress/theme/components/ArticleCard.vue -->
<template>
  <div class="card bg-base-100 shadow-xl">
    <figure v-if="post.frontmatter.cover">
      <img :src="post.frontmatter.cover" :alt="post.title" />
    </figure>
    <div class="card-body">
      <h2 class="card-title">
        <a :href="post.url">{{ post.title }}</a>
      </h2>
      <p>{{ post.frontmatter.description }}</p>
      <div class="card-actions justify-end">
        <span v-for="tag in post.frontmatter.tag" :key="tag" class="badge badge-primary">
          {{ tag }}
        </span>
      </div>
    </div>
  </div>
</template>

<script setup lang="ts">
  interface Props {
    post: {
      url: string
      title: string
      frontmatter: {
        cover?: string
        description?: string
        tag?: string[]
      }
    }
  }

  defineProps<Props>()
</script>

在 Markdown 中使用 ​

组件会自动注册，可以直接在 Markdown 中使用：

markdown
---
title: 我的文章
tag: [Vue, TypeScript]
---

# 文章列表

<ArticleCard 
  :post="{ 
    url: '/articles/demo', 
    title: '示例文章',
    frontmatter: { tag: ['Vue'] }
  }" 
/>

嵌套组件 ​

嵌套在 components/small/ 目录下的组件：

vue
<!-- .vitepress/theme/components/small/Icon.vue -->
<template>
  <span class="icon" :style="{ color: color }">
    <slot />
  </span>
</template>

<script setup lang="ts">
  withDefaults(defineProps<{ color?: string }>(), {
    color: 'currentColor',
  })
</script>

使用方式：small-Icon（路径中的 / 转换为 -）

Provide/Inject 跨组件通信 ​

在主题中提供数据 ​

typescript
// .vitepress/theme/index.ts
import { ref } from 'vue'

export default {
  enhanceApp({ app }) {
    const sharedState = ref({ theme: 'light' })
    app.provide('sharedTheme', sharedState)
  },
}

在组件中注入 ​

vue
<script setup lang="ts">
  import { inject } from 'vue'

  const sharedTheme = inject('sharedTheme')
</script>

总结 ​

  1. 数据获取：使用 createContentLoader 在构建时获取所有文章元数据
  2. 自动注册：import.meta.glob 自动发现并注册组件
  3. 数据流：从 content-loader.data.ts → theme/index.ts → 组件
  4. 组件引用：在 Markdown 中直接使用组件名，无需导入
